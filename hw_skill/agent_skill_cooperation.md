# Agent 与 Skill 配合工作机制

## 用户理解

> Agent 定义角色和权限，Skill 定义具体步骤，对吗？

**答案：基本正确，但不完全。。**

---

## Agent 的职责

Agent 是**角色定义**，包含：

| 维度 | 内容 |
|------|------|
| 角色定位 | 描述该 Agent 在系统中的职责（如"分析 Agent 负责读取原始数据并生成中文摘要"） |
| 权限模型 | 定义允许使用的工具（Read/Write/Edit/Glob 等）和禁止使用的工具 |
| 工作流程 | 描述执行步骤的逻辑（读取 → 分析 → 输出） |
| 输出格式 | 定义返回数据的结构 |
| 质量标准 | 定义自查清单 |

Agent 决定**"能做什么"**和**"不能做什么"**。

---

## Skill 的职责

Skill 是**具体执行步骤**，包含：

| 维度 | 内容 |
|------|------|
| YAML frontmatter | name、description（语义匹配用）、allowed-tools |
| 使用场景 | 描述何时使用该 Skill |
| 执行步骤 | 明确的执行步骤（1-7 步） |
| 注意事项 | 具体操作中的注意点 |
| 输出格式 | Skill 特有的输出格式 |

Skill 决定**"怎么做"**和**"按什么顺序做"**。

---

## 配合机制

```
┌─────────────────────────────────────────────────────────────────┐
│                         系统调度层                              │
│   (用户输入 "帮我采集 GitHub 热门项目")                          │
└─────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│                      语义匹配层                                  │
│   匹配 Skill：description 字段                                   │
│   匹配到：github-trending                                        │
└─────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Agent 约束层                                │
│   Skill 的 allowed-tools 限制 Agent 的工具权限                    │
│   github-trending Skill: Read/Glob/WebFetch/Grep               │
│   加载对应的 Agent（如 collector）并应用这些权限                  │
└─────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│                      执行层                                      │
│   Agent 按照 Skill 定义的具体步骤执行                            │
│   7 个步骤：搜索→提取→过滤→去重→摘要→排序→输出                    │
└─────────────────────────────────────────────────────────────────┘
```

---

## 关键区别

| 维度 | Agent | Skill |
|------|-------|-------|
| 本质 | **角色定义**（Who） | **步骤定义**（How） |
| 粒度 | 宏观（职责边界） | 微观（具体操作） |
| 复用性 | 同一 Agent 可被多个 Skill 调用 | 同一 Skill 对应特定场景 |
| 示例 | collector Agent 可以执行 github-trending skill | github-trending skill 只能采集 GitHub |

---

## 设计原则

1. **Agent 负责"能"**：定义权限边界，确保安全
2. **Skill 负责"做"**：定义具体流程，确保可执行
3. **两者正交**：Agent 关心权限，Skill 关心步骤，组合使用

---

## 示例：采集 GitHub Trending

| 层级 | 定义 |
|------|------|
| Agent（谁） | Collector Agent，权限：Read/Glob/WebFetch/Grep/Write |
| Skill（怎么做） | github-trending：7 个具体步骤 |
| 组合 | 加载 Collector Agent，应用 github-trending 的 allowed-tools，按 7 步骤执行 |

---

## 总结

| 问题 | 答案 |
|------|------|
| Agent 定义角色和权限？ | ✅ 是 |
| Skill 定义具体步骤？ | ✅ 是 |
| 两者如何配合？ | Skill 触发语义匹配 → Agent 应用 allowed-tools → 按 Skill 步骤执行 |
